# Design Boundary Review

## Verdict
`ReviseDesign`

## Human Summary
Revision 3 keeps the six typed contracts, exact cross-field source validation, bounded task construction, and durable request payload identity within CAP-D2. It also assigns ready-state compare-and-write, claim/task-exposure currentness transitions, stale-state effects, and transaction-race tests to CAP-D2 even though `workflowd-vs3.4.3`, `workflowd-vs3.4.4`, and `workflowd-vs3.4.7` explicitly own those runtime transitions. Revise the Design to leave CAP-D2 with typed payloads, validation functions, and narrow store/runner inputs, while those downstream tickets implement and test the lifecycle transitions.
